# anihub-cli — ROADMAP

> Цей файл описує новий функціонал для реалізації. Кожна фіча має чіткий опис поведінки, граничних випадків і зачіпок у коді.

## Контекст проєкту

TUI-клієнт для anihub.in.ua на Rust (ratatui + tokio + mpv IPC).

Структура модулів:
- `src/ui/app.rs` — `AppState`, `AppMode`, `FocusPanel`, обробка клавіш
- `src/ui/render.rs` — відмальовка TUI
- `src/main.rs` — точка входу, оркестрація
- `src/api/` — `client.rs`, `models.rs`, `grouper.rs`, `parser.rs`, `anilist.rs`
- `src/storage/history.rs` — `StorageManager`, `AppHistory`, `WatchProgress`
- `src/player/mod.rs` — `MpvPlayer`, IPC моніторинг

Поточна схема прогресу: `AppHistory.progress: HashMap<String, WatchProgress>` де ключ — `anime_id` як рядок. **Проблема:** один запис на аніме, сезон/серія перезаписується.

## Передумова — змінити ключ `WatchProgress`

Змінити ключ `HashMap` з `anime_id` на `"anime_id:season:episode"` щоб зберігати прогрес кожної серії окремо:

```rust
// Старий ключ:  "12345"
// Новий ключ:   "12345:2:7"  (anime_id:season:episode)

pub struct WatchProgress {
    pub anime_id: u32,
    pub anime_title: String,
    pub season: u32,
    pub episode: u32,
    pub timestamp: f64,   // секунди де зупинився
    pub duration: f64,    // загальна тривалість (0.0 якщо невідома)
    pub watched: bool,    // true якщо ≥ 80% або ≥ 20 хвилин
    pub updated_at: i64,
}
```

**Логіка `watched`** (рахується при збереженні):
```
if duration > 0.0 {
    watched = (timestamp / duration >= 0.80) || (timestamp >= 1200.0)
} else {
    watched = timestamp >= 1200.0
}
```

**Отримання `duration` з mpv IPC**: додати `observe_property` для `duration` поряд з `time-pos` у `monitor_ipc()`. Повертати `(time_pos, duration)` замість просто `f64`.

## Фіча 1 — Бібліотека

### Поведінка

- Клавіша `l` відкриває режим бібліотеки з будь-якого стану (окрім коли вже в бібліотеці — повторне натискання ігнорується).
- `Esc` на верхньому рівні бібліотеки → стартовий екран (порожній пошук, фокус на input).
- Стрілки вліво / `Backspace` — **не** повертають до стартового екрана.

### Відображення

- Аніме згруповані по назві — одна картка на франшизу.
- У кожній картці:
  - Назва аніме.
  - Прогрес у форматі `Сезон X · Серія Y · mm:ss` — остання переглянута позиція (найсвіжіша за `updated_at`).
  - Галочка `✓` якщо `watched == true`.
- Якщо кілька записів прогресу для одного аніме — у картці показується той що з найбільшим `updated_at`.

### Навігація

- Структура: Аніме → Сезон → Серія (дерево як у пошуку).
- `Esc` на рівні сезону/серії → рівень вище.
- `Esc` на верхньому рівні (список аніме) → стартовий екран.

### Зміни у `AppMode`

```rust
pub enum AppMode {
    // ... існуючі варіанти
    Library,        // верхній рівень — список аніме
    LibrarySeason,  // вибір сезону
    LibraryEpisode, // вибір серії
}
```

## Фіча 2 — Продовжити перегляд (`c`)

### Де працює

- На стартовому екрані (список порожній або щойно запустили).
- На конкретному аніме у результатах пошуку.
- На конкретному аніме в бібліотеці.

### Логіка на стартовому екрані

1. Знайти запис з найбільшим `updated_at` у `AppHistory.progress`.
2. Якщо `watched == false` → відкрити ту саму серію з позиції `timestamp`.
3. Якщо `watched == true` → знайти наступну серію того ж сезону. Якщо це остання серія — перша серія наступного сезону.
4. Якщо наступної серії/сезону нема → статус `"Усі серії переглянуто"`, нічого не запускати.
5. Виставити в `AppState` правильне аніме/сезон/серію/озвучку щоб UI показував контекст.
6. Запустити mpv з `start_time = timestamp` (або 0 для наступної серії).

### Логіка на конкретному аніме

- Те саме, але беремо найсвіжіший запис тільки для цього `anime_id`.
- Якщо запису нема → ігнорувати `c` (або статус `"Немає збереженого прогресу"`).

**Без підтвердження** — наступна серія запускається одразу.

## Фіча 3 — Статус знизу справа, помилки по центру

### Поведінка

- Прибрати центральні popup-и для **інформаційних** статусів: завантаження, "відтворюється в mpv", "пошук...", "завантаження серій..." тощо.
- Ці повідомлення → нижній правий кут (там де зараз шорткати).
- **Помилки** (мережа, mpv не знайдено, API error) → залишити по центру.

### Зміни у `AppState`

```rust
pub status_message: Option<(String, StatusKind)>,

pub enum StatusKind {
    Info,   // знизу справа
    Error,  // popup по центру
}
```

### Рендеринг (`render.rs`)

- `StatusKind::Info` → вписувати у нижній правий блок замість (або поряд з) шорткатами.
- `StatusKind::Error` → `Clear` + центрований `Paragraph` як зараз.
- Статус очищується через N секунд або при наступній дії користувача.

## Фіча 4 — Оптимізація пошуку (preload + кеш)

### Поточна проблема

Кожне переміщення стрілкою при навігації по результатах запускає новий HTTP-запит. Навігація повільна і залежна від мережі.

### Рішення: повний preload при Enter

При натисканні Enter у рядку пошуку:

1. Отримати список назв (`search_anime`) — як зараз.
2. **Одразу паралельно** запустити `get_episode_sources_for_anime` для **всіх** результатів через `tokio::spawn` + `FuturesUnordered` — щоб результати потрапляли в кеш по мірі надходження, не чекаючи всіх.
3. Результати зберігати у кеш в `AppState`:

```rust
pub episode_cache: HashMap<u32, EpisodeSourcesResponse>,
// ключ — anime_id
```

4. При навігації — брати з кешу. Якщо для поточного аніме ще не завантажилось → статус `"завантаження..."` знизу справа (Info), чекати.

### Кеш пошукових запитів

Зберігати результати попередніх пошуків щоб не робити повторний запит на той самий рядок:

```rust
pub search_cache: HashMap<String, Vec<AnimeSearchResult>>,
// ключ — trim().to_lowercase() від рядка запиту
```

- При Enter: спочатку перевірити `search_cache`. Якщо є — одразу показати результати і запустити preload тільки для тих `anime_id` яких немає в `episode_cache`.
- Кеш живе тільки в межах сесії, на диск не зберігається.

### Скасування запитів

При новому пошуку — скасовувати попередні фонові завантаження:

```rust
pub preload_abort: Option<tokio::task::AbortHandle>,
```

Або через `CancellationToken` з `tokio_util` якщо треба скасовувати батч з кількох spawn-ів.

### Для бібліотеки

- При відкритті `l` → запустити preload для всіх `anime_id` зі збереженого прогресу (тих що не в `episode_cache`).
- Фікс: при відображенні сезонів у бібліотеці зіставляти `studio.season_number` з `WatchProgress.season` а не брати перший-ліпший.

## Фіча 5 — Видалення з бібліотеки (`d`)

### Поведінка залежно від рівня

| Рівень | Що вибрано | Дія `d` |
|--------|------------|---------|
| Список аніме | Аніме | Видалити **весь** прогрес по `anime_id` (всі ключі `"anime_id:*"`) |
| Список сезонів | Сезон | Скинути `watched = false` для всіх серій сезону (timestamp зберігається) |
| Список серій | Серія | `watched == true` → скинути `watched`; `timestamp > 0` → обнулити `timestamp` |

### Підтвердження

- Видалення всього аніме (верхній рівень) → popup по центру: `"Видалити прогрес для [Назва]? [y/n]"`.
- Скидання сезону/серії → без підтвердження, одразу.

### Нові методи `StorageManager`

```rust
pub fn delete_anime_progress(&self, anime_id: u32) -> Result<()>
pub fn reset_season_watched(&self, anime_id: u32, season: u32) -> Result<()>
pub fn reset_episode_progress(&self, anime_id: u32, season: u32, episode: u32) -> Result<()>
```

## Фіча 6 — Esc до стартового екрана (з результатів пошуку)

### Поведінка

- `Esc` на верхньому рівні результатів пошуку (вибір між аніме) → очистити результати, стартовий екран.
- На нижніх рівнях (сезон, серія, озвучка) → `Esc` повертає на рівень вище.
- Стрілка вліво / `Backspace` — **не** повертають до стартового екрана.

### Зміни у `app.rs`

```rust
KeyCode::Esc => {
    match self.mode {
        AppMode::SearchResults => {
            self.results.clear();
            self.search_query.clear();
            self.mode = AppMode::Search;
        },
        AppMode::SeasonSelect | AppMode::EpisodeSelect => {
            self.go_up();
        },
        AppMode::Library => {
            self.mode = AppMode::Search;
        },
        _ => {}
    }
}
```

## Технічний борг (паралельно)

- Видалити невикористані залежності: `scraper`, `interprocess`.
- Змінити ключ `HashMap` в `AppHistory` (з `anime_id` на `anime_id:season:episode`) — **необхідна передумова** для фіч 1 і 2.
- `toggle_bookmark` — підключити до UI або видалити `#[allow(dead_code)]`.
- `generate_api_key` — прибрати хардкод секрету, читати з env або конфіг-файлу.
- Розбити `main.rs` (709 рядків) — логіка запуску відтворення переїжджає в `controller.rs` або `playback.rs`.
